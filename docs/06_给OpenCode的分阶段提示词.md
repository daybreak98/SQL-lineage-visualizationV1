# 06｜给 OpenCode 的分阶段提示词

---

## 1. 总控提示词

```text
你现在负责实现 SQL 血缘解析可视化项目，但必须严格按 C00-C10 分轮推进。
每一轮只允许实现对应文档中的能力，不允许提前实现后续轮次。
每轮完成后必须：
1. 列出修改文件；
2. 说明后端 API 如何验收；
3. 说明前端页面如何看到效果；
4. 运行对应 pytest；
5. 明确是否满足进入下一轮条件。
如果发现现有代码与文档冲突，以当前轮次文档为准。
不要通过删除测试、硬编码 fake success、注释前端错误来过关。
```

---

## 2. C00-C10 单轮提示词索引

每轮文档末尾都包含专用提示词。使用方式：

```text
请严格按照 05_每轮开发文档/Cxx_xxx.md 实现。
只做 Cxx，不做 Cxx+1。
```

---

## 3. 每轮完成后的复盘提示词

```text
请审查刚才完成的 Cxx 实现是否越界。
重点检查：
1. 是否提前实现了后续轮次能力；
2. 是否破坏了 AnalysisResult 契约；
3. 是否存在 edge.source / edge.target 悬空；
4. 是否有 unknown / ambiguous 被伪装成 success；
5. 是否有前端自行推断血缘；
6. 测试是否真实覆盖，而不是只测空响应。
请给出问题清单和必须修复项。
```

---

## 4. 调试失败提示词

```text
当前 Cxx 实现失败。请按以下顺序排查：
1. 后端是否能 uvicorn 启动；
2. API 路径是否与前端一致；
3. 请求体是否被 Pydantic 正确接收；
4. 响应 JSON 是否满足 03_后端最小契约；
5. 前端 Network 是否请求 /api/...；
6. Vite proxy 是否转发到 8000；
7. graph_view_model 的 node / edge 是否结构完整；
8. diagnostics 是否被正确展示。
不要新增无关功能，只修复当前轮次失败点。
```

---

## 5. 前端联调提示词

```text
请根据当前轮次文档检查前端联调。
要求：
1. 前端请求必须使用 /api 相对路径；
2. 不允许在前端硬编码后端地址；
3. 前端不允许推断 SQL 血缘；
4. 前端只消费后端返回的 output_fields、graph_view_model、diagnostics_report、source_locations；
5. 当前轮次必须有一个页面可见效果。
请输出需要改动的前端文件和验收方式。
```
