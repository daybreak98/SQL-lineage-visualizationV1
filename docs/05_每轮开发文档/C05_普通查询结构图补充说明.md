# C05 补充：普通查询结构图返回

## 本次补充实现了什么

普通查询现在也会返回结构图。

单表 SQL：

```sql
select a from t
```

返回结构边：

```text
physical_table:t -> query_result:final
```

Join SQL：

```sql
select u.country_name, o.order_no
from dim_user_df u
join dwd_order_di o on u.user_id = o.user_id
```

返回结构边：

```text
physical_table:dim_user_df -> query_result:final
physical_table:dwd_order_di -> query_result:final
```

同时继续保留字段血缘边：

```text
physical_column:dim_user_df.country_name -> output_column:country_name
physical_column:dwd_order_di.order_no -> output_column:order_no
```

## 为什么这样做

结构图和字段图是两种视角，不应该按 SQL 类型互斥：

```text
Table 视图：表 / CTE / Query Result 的结构依赖
Column 视图：字段到输出字段的血缘关系
```

本次不做大重构，只在同一个 `graph_view_model` 中同时返回结构节点和字段节点。前端根据视图模式过滤展示。

## 开发路径

1. 新增 `table_structure_service.py`，负责普通 SQL 的表级结构依赖。
2. 扩展 `graph_builder.py`，支持 `build_table_structure_graph` 和 `merge_graphs`。
3. 修改 Analyze API：
   - CTE SQL 继续返回 CTE 结构图。
   - 普通 SQL 返回结构图 + 字段图的组合图。
   - 默认 `graph_view_model.view_mode = table`。
4. 修改前端：
   - Analyze 后默认进入 Table 视图。
   - Table 视图只展示结构节点和结构边。
   - Column 视图展示字段节点和字段边。
5. 补充后端和前端测试。

## 当前边界

本次仍不做多视图契约重构。

也就是说，暂时没有新增：

```text
graph_views
available_views
独立 structure_graph / column_graph 字段
```

当前策略是：一个 `graph_view_model` 同时携带结构节点和字段节点，由前端视图过滤展示。
