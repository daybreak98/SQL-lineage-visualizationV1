# C05 补充：表级视图紧凑重布局说明

## 本次修复了什么

修复 Table 视图中 `Query Result` 距离表节点过远的问题。

之前 Table 视图只是从 Subquery 视图中过滤掉 CTE / 子查询节点，但节点坐标仍沿用 Subquery 图坐标，所以最终节点会停留在子查询链路最右侧。

现在 Table 视图会按当前可见节点重新计算布局。

## 当前规则

```text
Table 视图：
左侧：物理表节点
右侧：Query Result
边：物理表 -> Query Result
```

结果节点的纵坐标会根据表节点数量居中。

## 为什么这样做

每个视图都应该按自己展示的节点构造布局。

表级血缘只展示物理表和最终结果时，不应该继承子查询级血缘的远距离坐标。
