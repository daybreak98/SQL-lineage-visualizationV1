# C08 补充：字段血缘汇总到最终查询节点修复说明

## 问题

字段级血缘图之前只表达：

```text
physical_column -> output_column
```

例如：

```text
dwd_order_di.order_no -> order_no
dwd_order_di.user_id  -> uid
```

但没有继续表达：

```text
output_column -> Query Result
```

所以多个输出字段在字段级视图里没有汇总到最终查询节点上。前端即使拿到了结构图里的 `query_result:final`，列血缘视图也会把它过滤掉，导致视觉上看不到字段最终流向查询结果。

## 根因

后端 `build_column_lineage_graph()` 只创建了物理字段节点、输出字段节点和 `column_lineage` 边，没有创建最终查询节点，也没有创建输出字段到最终查询的边。

前端 `visibleColumnGraph()` 原本只允许显示：

```text
column
output_field
expression
unknown
```

因此字段级视图无法展示 `query_result:final`。修复后又发现，如果直接沿用旧的列布局，输出字段和最终查询节点之间距离太近，边段很短，视觉上仍然像没有真正连接。

## 修复

后端修改：

```text
backend/app/services/graph_builder.py
```

字段图现在会补充：

```text
query_result:final
output_column:<name> -> query_result:final
```

新增边类型：

```text
output_column_to_result
```

前端修改：

```text
frontend/src/graphPipeline.ts
```

新增边类型映射：

```text
output_column_to_result -> output
```

Column 视图现在允许展示 `output` 节点，但只有存在 `output_field` 时才展示最终查询节点。这样普通字段图可以看到最终 `Query Result`，而 CTE 没有字段节点时不会只剩一个孤立的查询结果节点。

同时，Column 视图改为按当前可见节点重新计算布局：

```text
source column -> output field -> Query Result
```

不再复用子查询结构图的远距离位置。当前列血缘的横向间距固定为更均衡的三段式布局，保证字段节点到最终查询节点之间有可读的边长，也避免 Query Result 被推到屏幕外。

## 现在字段级结构

修复后字段级图表达为：

```text
physical_column -> output_column -> Query Result
```

例如：

```text
dwd_order_di.order_no -> order_no -> Query Result
dwd_order_di.user_id  -> uid      -> Query Result
```
