# C12 前端冗余代码与搜索过滤清理说明

## 本轮目标

清理前端中遗留的冗余代码、异常文案，并补充搜索过滤测试，降低后续维护成本。

## 主要改动

### 1. 删除 review-only 源文件

从 `frontend/src` 删除两份评审包代码：

- `lineage-graph-pipeline--for-review.tsx`
- `lineage-graph-v2--for-review.tsx`

这两份文件不被运行时代码引用，却被 TypeScript 纳入 `src` 编译范围；其中一份还含有非法 UTF-8 字节。删除后，前端源码目录只保留实际应用代码和测试 fixture。

### 2. 精简 mockLineage fixture

`frontend/src/data/mockLineage.ts` 现在只保留测试和 Drawer 说明面板仍需要的数据：

- `subqueryNodes`
- `subqueryEdges`
- `snapshots`
- `milestones`

删除了旧的 mock entities、mock mappings、mock search outputs、mock paths，避免运行时代码继续依赖早期假数据。

### 3. 修复搜索 scope 过滤

`SearchBar` 原先只有 `output` scope 会过滤，`source / cte / subquery / expression` 下拉选项看起来可选，但不会真正改变结果。

现在统一按两层条件过滤：

- query 命中 display/sub/reason
- scope 命中 item type；其中 `cte` scope 按 `entityId` 的 `cte:` 前缀识别

同时修复搜索胶囊里的异常分隔符。

### 4. 清理重复配置

删除 `frontend/package.json` 中重复定义的 `test` / `test:watch` script key，保留唯一脚本定义。

## 测试

新增 SearchBar scope 回归测试：

- `source` 只显示 source 结果
- `cte` 只显示 CTE entity
- `expression` 只显示 expression 结果

验证命令：

```powershell
npm test -- --run
npm run typecheck
npm run build
```

结果：

```text
Vitest: 7 files passed, 97 tests passed
TypeScript: passed
Vite build: passed
```
