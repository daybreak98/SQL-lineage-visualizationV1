# C05 本轮实现说明

## 本轮实现了什么

C05 实现了 CTE / 子查询结构依赖图。

对于如下 SQL：

```sql
with order_base as (
  select user_id, order_no, order_amount
  from dwd_order_di
),
metric_base as (
  select user_id, count(order_no) as order_cnt, sum(order_amount) as gmv
  from order_base
  group by user_id
)
select user_id, order_cnt, gmv
from metric_base
```

后端现在返回结构依赖：

```text
dwd_order_di -> order_base -> metric_base -> Query Result
```

并且 `graph_view_model.view_mode` 返回：

```text
subquery_dependency
```

## 开发路径

1. 阅读 C05 文档，确认本轮只做结构依赖图，不做完整字段穿透、SourceLocation、Semantics、SQLite metadata。
2. 用 SQLGlot 确认 CTE 的 AST 结构：`tree.args["with"].expressions` 是 CTE 列表，`cte.alias` 是 CTE 名称，`cte.this.find_all(Table)` 能找到上游依赖。
3. 新增 `cte_structure_service.py`，负责抽取物理表、CTE、Query Result 之间的结构依赖。
4. 扩展 `graph_builder.py`，新增 `build_cte_structure_graph`，生成 `view_mode=subquery_dependency` 的 GraphViewModel。
5. 修改 Analyze API：`WITH` SQL 走 CTE 结构图路线；普通 SQL 继续走 C04 字段血缘路线。
6. 新增 C05 单元测试和 API 集成测试。

## 本轮边界

本轮不做：

```text
完整复杂子查询字段穿透
SourceLocation
SemanticsReport
SQLite 元数据导入
select * 展开
```

```text
nodes 包含 physical_table:dwd_order_di
nodes 包含 cte:order_base
nodes 包含 cte:metric_base
nodes 包含 query_result:final
edges 包含 dwd_order_di -> order_base
edges 包含 order_base -> metric_base
edges 包含 metric_base -> Query Result
CTE 节点 node_type 必须是 cte，不能伪装成 physical table
```
