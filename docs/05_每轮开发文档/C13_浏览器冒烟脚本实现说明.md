# C13 浏览器冒烟脚本实现说明

## 目标

之前用浏览器插件验收字段级血缘时，需要在对话里执行很多步：

```text
打开页面 -> 设置 SQL -> 点击 Analyze -> 切 Column -> 读 DOM -> 截图
```

这种方式准确，但 token 成本高。现在把这套流程沉淀成项目脚本，日常只输出结构化 JSON，只有需要视觉验收时才保存截图。

## 新增文件

```text
frontend/e2e/lineage_browser_smoke.py
frontend/e2e/cases/single_table_column.sql
```

并在 `frontend/package.json` 增加命令：

```text
npm run smoke:browser
```

## 默认验收内容

默认 SQL：

```sql
select
  order_no,
  user_id as uid
from dwd_order_di
```

默认验证 Column 字段级视图：

```text
1. 页面可打开。
2. Monaco 编辑器能被写入 SQL。
3. Analyze 能触发真实前端流程。
4. Column 视图可切换。
5. DOM 中出现 5 个节点：
   - dwd_order_di.order_no
   - order_no
   - dwd_order_di.user_id
   - uid
   - Query Result
6. DOM 中出现 4 条可视边：
   - 2 条字段投影边
   - 2 条输出字段汇总到 Query Result 的边
7. API 中出现 5 条边：
   - 1 条表级边
   - 4 条字段级边
```

## 使用方式

普通冒烟：

```powershell
cd frontend
npm run smoke:browser
```

保存截图：

```powershell
cd frontend
python e2e/lineage_browser_smoke.py --screenshot C:/Users/Administrator/AppData/Local/Temp/lineage-smoke.png
```

指定 SQL 文件：

```powershell
cd frontend
python e2e/lineage_browser_smoke.py --sql-file e2e/cases/single_table_column.sql
```

追加期望节点：

```powershell
cd frontend
python e2e/lineage_browser_smoke.py --expect-node "Query Result"
```

## 设计原则

这个脚本保留在项目内，而不是做成 skill。原因是它依赖本项目的页面结构、按钮文案、Monaco 编辑器、`/api/sql/analyze` 接口、节点 class 和边 class。

后续如果多个项目都复用这类模式，再把方法论提炼成 skill 会更合适。
