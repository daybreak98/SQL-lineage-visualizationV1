# C03｜单表字段血缘与画布首图

## 1. 本轮目标

实现最简单、最有成就感的一张血缘图：

```sql
select a, b as bb from t
```

生成：

```text
t.a  → output.a
t.b  → output.bb
```

```text
后端能力：单表简单字段血缘
前端效果：画布出现第一组节点和边
学习重点：最小 GraphViewModel、node id、edge source/target 校验
```

---

backend/app/domain/graph_view_model.py

---

## 3. 支持范围

只支持：

```sql
select a from t
select t.a from t
select a as aa from t
select a, b from db.table
```

暂不支持：

```text
join
cte
subquery
select *
复杂表达式来源
同名字段冲突
```

---

## 4. Graph 响应样例

```json
"graph_view_model": {
  "view_mode": "column",
  "nodes": [
    { "id": "physical_column:t.a", "node_type": "physical_column", "label": "t.a" },
    { "id": "output_column:a", "node_type": "output_column", "label": "a" }
  ],
  "edges": [
    {
      "id": "edge:physical_column:t.a->output_column:a",
      "source": "physical_column:t.a",
      "target": "output_column:a",
      "edge_type": "column_lineage"
    }
  ]
}
```

---

## 5. 前端对接文档

前端 Analyze 后：

```text
1. 如果 nodes / edges 非空，画布渲染图
2. 节点标签使用 label
3. 线条使用 source / target 连接
4. 点击 output 字段时，可高亮相关边
```

本轮前端不要求复杂布局，只要能看到节点和边。

---

## 6. 测试验收

后端测试：

```text
select a from t
- nodes 包含 physical_column:t.a
- nodes 包含 output_column:a
- edges 包含 t.a -> output.a
- edge.source / target 都存在
```

前端验收：

```text
输入 select a from t
点击 Analyze
画布出现两个节点和一条边
```

---

## 7. 禁止越界

不要在本轮处理 Join / CTE。遇到多表 SQL 可以返回 partial + UNSUPPORTED_COMPLEX_QUERY。
