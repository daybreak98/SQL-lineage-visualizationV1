# C05 补充：画布结构布局修复说明

## 本次修复了什么

修复 CTE 结构图中 `Query Result` 节点出现在左侧的问题。

修复后，结构图按依赖层级从左到右排列：

```text
physical table -> CTE / subquery -> final CTE / subquery -> Query Result
```

也就是说，最终查询结果节点永远放在当前结构血缘链路的最右侧。

## 为什么之前会错

后端已经返回了 `query_result:final` 节点。

前端看到后端已经有结果节点，就不会再创建新的 `Query Result` 节点。但旧布局逻辑只给表和 CTE 计算依赖层级，没有给 output 节点计算层级，所以 output 节点会落回默认坐标，出现在左侧。

## 本次怎么修

1. 前端只对结构节点做结构图布局。
   - `table`
   - `cte`
   - `subquery`
   - `output`

2. 字段节点不参与结构图层级布局。
   - `physical_column`
   - `output_column`

3. `output` 节点统一放在最大结构依赖层级之后。

4. `output` 节点的纵坐标参考它的直接上游节点，使最终结果更靠近主血缘路径。

5. 画布边的贝塞尔曲线改成动态控制点，边不再过直，长边和短边都会自然弯曲。
