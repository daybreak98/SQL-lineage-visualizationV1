# C05 补充：表级与子查询视图拆分说明

## 本次修复了什么

本次把画布里的两个概念拆清楚：

```text
Subquery 视图：物理表 -> CTE / 子查询 -> Query Result
Table 视图：物理表 -> Query Result
```

这样 CTE 不会再混进表级视图里。

## 为什么要拆

表级血缘关注的是“最终结果依赖哪些物理表”。

子查询血缘关注的是“SQL 内部的中间结构怎样流转”。

如果把两者混在一个视图里，就会同时看到：

```text
表 -> CTE -> 结果
表 -> 结果
```

这会让画布出现多余捷径边，看起来很乱。

## 本次怎么做

1. 新增前端视图模式 `subquery`。
2. CTE 分析结果默认进入 `Subquery` 视图。
3. `Table` 视图只展示 `table` 和 `output` 节点。
4. 原始结构图不再自动补“所有结构节点 -> Query Result”的边。
5. `Table` 视图如果过滤掉了 CTE，会在选择器层临时展示 `table -> output`，但不污染原始后端图。
6. 节点字体和框体缩小到约 80%。
7. 节点框内不再显示 `CTE / SUBQ / OUT` 标签，改用颜色和边框区分类型。
8. 清理了前端源码里的乱码注释和可见乱码分隔符。
