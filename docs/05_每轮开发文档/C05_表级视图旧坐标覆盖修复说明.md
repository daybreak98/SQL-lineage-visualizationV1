# C05 补充：表级视图旧坐标覆盖修复说明

## 本次修复了什么

修复 Table 视图里 `Query Result` 仍然距离表节点很远的问题。

之前虽然 Table 视图已经重新计算了紧凑坐标，但 Analyze 成功后会把原始后端图坐标写入 `state.positions`。

`LineageCanvas` 渲染时会优先使用 `state.positions`，所以 Table 视图的紧凑坐标被旧的 Subquery 坐标覆盖了。

## 本次怎么修

1. Analyze 成功后不再把默认图坐标写入 `positions`。
2. `positions` 只保留用户手动拖拽后的节点位置。
3. 切换视图时清空 `positions`，确保每个视图使用自己的布局。
4. 补充 UI 回归测试，验证 Table 视图里最终节点距离表节点不会过远。
5. 清理开发中遇到的乱码文案。

## 当前规则

```text
默认布局坐标：来自 visibleGraph 当前视图
用户拖拽坐标：来自 state.positions
视图切换：清空用户拖拽坐标，重新使用当前视图布局
```
