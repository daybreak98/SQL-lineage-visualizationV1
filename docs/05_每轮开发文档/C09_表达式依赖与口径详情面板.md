# C09｜表达式依赖与口径详情面板

## 1. 本轮目标

让系统不仅知道字段从哪来，还能解释指标怎么算。

```sql
select
  sum(order_amount) as gmv,
  count(distinct order_no) as order_cnt,
  sum(order_amount) / count(distinct order_no) as adr
from dwd_order_di
```

目标：

```text
adr 依赖 order_amount 和 order_no
gmv 是 sum(order_amount)
order_cnt 是 count(distinct order_no)
DetailPanel 能展示表达式和依赖字段
```

---

## 2. 后端能力

新增：

```text
expression node
expression_dependency edge
semantics_report 基础字段
指标表达式文本
聚合函数识别
```

---

## 3. 响应样例

```json
"semantics_report": {
  "metrics": [
    {
      "name": "adr",
      "expression": "SUM(order_amount) / COUNT(DISTINCT order_no)",
      "depends_on": ["dwd_order_di.order_amount", "dwd_order_di.order_no"],
      "description": "平均订单金额，由 GMV 除以订单数得到。"
    }
  ]
}
```

---

## 4. 前端对接文档

点击 output 或 expression 节点后，底部 DetailPanel 展示：

```text
字段名
表达式
上游字段
聚合函数
诊断信息
字段注释
```

注意：解释文本不是血缘证据，血缘证据仍然来自 edges。

---

## 5. 测试验收

```text
sum(order_amount) as gmv → depends_on 包含 order_amount
count(distinct order_no) as order_cnt → depends_on 包含 order_no
gmv / order_cnt as adr → depends_on 包含 gmv/order_cnt 或其物理字段来源
表达式节点 edge.source / target 不悬空
```

---

## 6. 禁止越界

不要做自然语言大模型解释，不要接 AI。C09 只做基于 SQL 结构的确定性解释。
