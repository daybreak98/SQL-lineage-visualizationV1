# C09 表达式依赖：退化字段依赖支持说明

## 当前决策

C09 不继续开发完整的“表达式口径详情面板”和完整 expression 图语义。

但后端需要支持一个退化能力：

```text
从表达式 SELECT 项中提取源字段，并建立 源字段 -> 输出字段 -> Query Result 的列级血缘。
```

也就是说，系统暂时不解释“怎么算”，但要知道“依赖哪些字段”。

## 支持范围

示例 SQL：

```sql
SELECT
  SUM(order_amount) AS gmv,
  COUNT(DISTINCT order_no) AS order_cnt,
  SUM(order_amount) / COUNT(DISTINCT order_no) AS adr
FROM dwd_order_di
```

当前应解析出输出字段：

```text
gmv
order_cnt
adr
```

以及字段依赖：

```text
dwd_order_di.order_amount -> gmv
dwd_order_di.order_no     -> order_cnt
dwd_order_di.order_amount -> adr
dwd_order_di.order_no     -> adr
```

每个输出字段继续汇总到最终查询节点：

```text
gmv       -> Query Result
order_cnt -> Query Result
adr       -> Query Result
```

## 不支持范围

当前不要求：

```text
1. 生成 expression 节点。
2. 生成 expression_dependency / expression_to_output 语义边。
3. 生成完整 semantics_report.metrics。
4. 在 DetailPanel 展示聚合函数、distinct、除法口径等解释。
5. 判断 adr 是 gmv / order_cnt 还是直接依赖底层物理字段后的派生指标。
```

这些仍属于后续完整 C09。

## 实现位置

核心逻辑在：

```text
backend/app/services/name_resolver.py
```

当 SELECT 项不是简单字段投影时，resolver 会：

```text
1. 遍历表达式内部的 sqlglot exp.Column。
2. 根据表别名和元数据规则解析字段所属表。
3. 对同一输出字段去重相同源字段。
4. 生成 SimpleColumnLineage。
```

图构建仍复用已有字段血缘图：

```text
backend/app/services/graph_builder.py
```

所以输出图仍是：

```text
physical_column -> output_column -> Query Result
```
