# C14 字段级血缘表节点上游展示说明

## 问题

字段级血缘的 Column 视图之前直接展示物理字段节点：

```text
dwd_order_di.order_no -> order_no -> Query Result
dwd_order_di.user_id  -> uid      -> Query Result
```

这样虽然字段证据精确，但视觉上最上游是一组 `表名.字段名` 节点，不符合“血缘最上游应该先看到表”的阅读习惯。

## 调整

本轮只调整前端 Column 视图的可视层。

后端 GraphViewModel 仍保留精确证据：

```text
physical_column:dwd_order_di.order_no -> output_column:order_no
physical_column:dwd_order_di.user_id  -> output_column:uid
```

前端展示时把物理字段节点折叠到所属表节点：

```text
dwd_order_di -> order_no -> Query Result
dwd_order_di -> uid      -> Query Result
```

这样既保留后端字段依赖证据，又让画布最上游从表节点开始。

## 实现位置

```text
frontend/src/graphPipeline.ts
```

核心逻辑：

```text
1. Column 视图不再直接显示 physical_column 节点。
2. 对 column_lineage 边做可视层折叠：
   physical_column:<table>.<column> -> output_column:<name>
   转成：
   physical_table:<table> -> output_column:<name>
3. 移除 Column 视图中的 table_to_result 直连，避免表节点同时直连 Query Result 造成图变乱。
4. output_column -> Query Result 保持不变。
```
