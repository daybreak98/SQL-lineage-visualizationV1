# C05 补充：画布连线与默认居中修复说明

## 本次修复了什么

本次继续优化血缘画布的可读性：

1. 去除边上的文字标签。
2. 短边改为直线。
3. 长边保留轻微贝塞尔曲线。
4. SQL 解析完成后，画布默认把当前可见血缘图居中展示。
5. 开发过程中遇到乱码注释或明显乱码文案时同步清理。

## 为什么这样做

血缘图的核心目标是快速看清依赖方向。

边上再写 `table`、`cte`、`output` 会重复节点和颜色已经表达的信息，反而增加视觉噪音。

曲线也不是越多越好。短距离依赖使用直线更清楚；长距离依赖使用轻微曲线，可以减少穿插时的压迫感。

## 本次怎么做

1. `LineageCanvas` 不再渲染 `.edge-label` 文本节点。
2. 删除不再使用的 `.edge-label` CSS。
3. 边路径规则改为：

```text
短边：M x1 y1 L x2 y2
长边：M x1 y1 C c1x c1y, c2x c2y, x2 y2
```

4. 画布根据当前可见节点的外接矩形和 viewport 尺寸计算 `translate(x, y)`。
5. 自动居中基于原始布局坐标计算，用户拖拽节点时不会导致整张图重新漂移。
