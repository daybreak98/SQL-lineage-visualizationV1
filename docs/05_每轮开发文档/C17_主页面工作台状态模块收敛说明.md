# C17 主页面工作台状态模块收敛说明

日期：2026-06-11  
阶段：已实现  
状态：本轮完成第 1 小点

## 本轮目标

根据《主页面 SQL 血缘工作台架构优化方案》，先完成第一步最小重构：

- 不改主页面交互行为
- 先把 `App.tsx` 中最核心的工作台状态与状态迁移逻辑抽到独立模块
- 为后续 action / reducer 化做准备

## 本轮范围

### 包含

- 新增 `frontend/src/workbench/state.ts`
- 抽离以下逻辑：
  - 初始 `WorkbenchState`
  - backend diagnostic 归一化
  - SQL 编辑后的脏状态迁移
  - analyze 运行中状态
  - analyze 成功态归并
  - analyze 失败态归并
  - 搜索结果选择后的状态迁移
- 新增状态模块测试

### 不包含

- 不改 `LineageCanvas / DetailPanel / CanvasToolbar / Drawer` 的状态写法
- 不引入 reducer
- 不拆 `graphPipeline.ts`
- 不清理 `selectors.ts` 中的 mock / 夹杂逻辑

## 实现内容

### 1. 新增 Workbench 状态模块

新增：

- `initialWorkbenchState`
- `normalizeWorkbenchDiagnostic(...)`
- `applySqlDraftChange(...)`
- `buildAnalyzeRunningState(...)`
- `buildAnalyzeSuccessState(...)`
- `buildAnalyzeFailureState(...)`
- `applySearchSelection(...)`

这样处理后：

- `App.tsx` 不再内联维护整段工作台初始状态
- 分析成功 / 失败和搜索结果选择不再散落在页面主体内部

### 2. App.tsx 改为复用状态模块

`App.tsx` 当前保留：

- 页面级状态持有
- 顶层请求触发
- 顶层组件装配

`App.tsx` 当前移出：

- Workbench 初始状态定义
- backend diagnostic 归一化实现
- analyze 成功态归并细节
- analyze 失败态归并细节
- SQL 编辑变脏规则
- 搜索结果选择后的状态迁移细节

## 本轮收益

- `App.tsx` 职责边界更清楚
- 核心工作台状态迁移可以被单测直接覆盖
- 为下一轮收敛 action 提供稳定落点
