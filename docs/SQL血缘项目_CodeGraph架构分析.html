五、complex_sql_guard 防御解析引擎（★新增核心模块★）

独立于 app 其他模块的防御性 SQL 解析引擎。支持对脏 SQL（模板变量、正则函数、Freemarker 块、长 SQL）进行多阶段预处理后再解析，最大化解析成功率。

阶段1：Preflight（预检）

文件：preflight.py
核心类：PreflightChecker
检查项：SQL大小(>50K)、复杂度(JOIN/UNION/子查询)、括号平衡、引号平衡、模板检测、隐式字符检测

阶段2：Shields（净化）

文件：shields.py
核心类：DirtySqlPreprocessor
替换：注释(单行/块)、Hint(/*+)、字符串字面量、模板变量(${}/#{}）、Freemarker块(<#>)、反引号标识符

阶段3：Segmenter（分句）

文件：segmenter.py
核心类：SqlSegmenter
按关键词切分：WITH→CTE、SELECT→列、FROM→源表、JOIN→连接块、WHERE→条件、GROUP/ORDER/HAVING/UNION

阶段4：Parse（解析）

文件：parser_adapter.py
核心类：SqlglotParserAdapter
3 个候选依次尝试：①原始SQL ②规范化SQL ③净化后SQL。任一成功即停止

阶段5：Segment Parse（回退）

文件：analyzer.py
全量解析失败时，为每个子句(SELECT列、WHERE条件、JOIN条件)套最小SQL包装后逐一尝试解析

阶段6：AST Walk（走查）

文件：analyzer.py
解析成功后走查 AST：检测 LateralView(lateral视图)、Anonymous函数(黑盒UDF)、行展开函数(explode等)

设计要点：该模块完全独立于 app 其他模块（不 import app.services/domain/controllers），仅依赖 sqlglot 库。
输出：ComplexSqlAnalysisResult（含 text_bundle、segments、parse_attempts、diagnostics、capabilities、confidence）
桥接层：sql_parse_service.py 调用 analyze_complex_sql()，将结果翻译为 ParseServiceResult

十一、关键诊断码索引

诊断码级别含义触发条件
SQL_PARSE_ERROR error SQL 解析失败 sqlglot.parse_one() 抛出 ParseError
UNKNOWN_COLUMN warning 字段不在元数据中 表在 metadata 中，但字段不在其列列表里
AMBIGUOUS_COLUMN warning 多表有同名字段 未限定的字段出现于 ≥2 张表
METADATA_MISSING warning 表不在元数据中 SQL 引用的表在 SQLite 元数据中查不到
SELECT_STAR_METADATA_REQUIRED warning select * 需要元数据 SQL 中有 *，但元数据不可用
UNKNOWN_TABLE_ALIAS warning 表别名无法解析 列限定符不在 alias_to_table 映射中
UNSUPPORTED_COMPLEX_QUERY warning 复杂表达式暂不支持 SELECT 中有非 Column/Alias 的复杂AST节点
SOURCE_LOCATION_APPROXIMATE info 位置是近似的 select * 展开时所有列共享 * 的位置
