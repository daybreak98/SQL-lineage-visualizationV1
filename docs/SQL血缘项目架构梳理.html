一、目录结构与分层

层目录职责禁止做的事
Controllerapi/收 HTTP 请求、调 Service、返回响应不写业务逻辑
Serviceservices/编排领域逻辑、调用 Adapter/Repository不直接操作 SQL/HTTP
Domaindomain/纯数据结构定义，零外部依赖不引任何第三方库
Adapteradapters/封装第三方库，翻译为内部类型不写业务规则
Repositoryrepositories/SQL 读写，数据库操作不写业务判断
Guardcomplex_sql_guard/防御性 SQL 解析引擎（独立模块）不依赖 app 其他模块
DBdb/数据库连接 + 迁移脚本—

controller 层文件：

文件端点功能
health_controller.pyGET /api/health健康检查
analyze_controller.pyPOST /api/sql/analyzeSQL 血缘分析（核心入口）
metadata_controller.py4 个端点元数据导入/查询

service 层文件：

文件功能
sql_parse_service.py解析入口，桥接 complex_sql_guard，生成 output_fields + diagnostics
name_resolver.py字段→表归属 + alias 解析 + metadata 字段验证/消歧 + star 委托
star_expansion_service.pyAST 级 select * 展开（exp.Star / Column(this=Star)）
table_structure_service.py物理表→Query Result 结构边（table_to_result）
cte_structure_service.pyCTE 层级依赖图（table_to_cte → cte_dependency → cte_to_result）
graph_builder.py图组装（build_*_graph）+ 合并（merge_graphs）+ 边校验（validate_graph）
metadata_import_service.py元数据 preview/commit 业务规则（重复版本跳过）

complex_sql_guard 内部文件：

文件阶段功能
analyzer.py主编排6 阶段 pipeline 总调度，决定解析结果和置信度
preflight.py1. 预检SQL 大小/复杂度/引号平衡/括号平衡/隐式字符检查
shields.py2. 净化替换模板变量/字符串字面量/注释/hint/Freemarker 为占位符
segmenter.py3. 分句按 WITH/SELECT/FROM/WHERE/JOIN 等子句切分
parser_adapter.py4/5. 解析3 个候选项（原始/规范化/净化后）依次尝试；失败后按子句解析
normalizer.py辅助换行符规范化 + 偏移映射（分析文本↔原始文本定位）
dialect.py辅助各 SQL 方言的引号规则/函数注册表/UDTF 列表
diagnostics.py辅助模块内诊断码常量（22 个）

二、血缘解析全流程

POST /api/sql/analyze
  │
  ▼
┌─ analyze_controller.py ──────────────────────────────────────┐
│                                                               │
│  1. parse_sql(sql, dialect)                                  │
│     └─→ complex_sql_guard（6 阶段防御解析）                     │
│         ├─ preflight:    大小/复杂度/引号/括号检查               │
│         ├─ shields:      模板/字面量/注释 替换为占位符           │
│         ├─ segmenter:    SQL 分句 (WITH/FROM/WHERE..)        │
│         ├─ parser:       3 候选依次尝试解析                    │
│         │   ├─ 原始 SQL                                      │
│         │   ├─ 规范化 SQL                                    │
│         │   └─ 净化后 SQL                                    │
│         ├─ segment parse: 全量失败时按子句解析                  │
│         └─ tree diagnostics: AST 走查 (UDF/lateral)          │
│     └─→ 返回 tree + output_fields + diagnostics              │
│                                                               │
│  2. 判断 CTE / 非-CTE                                        │
│                                                               │
│  ┌── CTE 分支 (sql.startswith("with")) ──────────────────┐  │
│  │ analyze_cte_structure(tree=tree)                      │  │
│  │  → physical → cte → cte → result 层级图               │  │
│  │ build_cte_structure_graph()                           │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                               │
│  └── 非-CTE 分支 ────────────────────────────────────────┐  │
│     _extract_source_table_names(tree)                   │  │
│     _load_metadata(table_names) ← SQLite                │  │
│                                                         │  │
│     analyze_table_structure(tree=tree)                  │  │
│      → physical_table → query_result 边                 │  │
│                                                         │  │
│     resolve_column_lineage_names(tree, metadata)        │  │
│      │                                                  │  │
│      ├─ 有 * + 有元数据                                   │  │
│      │  └─ star_expansion_service 展开                   │  │
│      │     └─ 查 metadata → 生成 lineage                 │  │
│      │                                                  │  │
│      ├─ 有限定字段 (u.country_name)                       │  │
│      │  └─ alias_to_table 解析 → 归属物理表               │  │
│      │                                                  │  │
│      ├─ 无限定 + 单表 → 直接归属                          │  │
│      │                                                  │  │
│      ├─ 无限定 + 多表 + 元数据齐全                         │  │
│      │  ├─ 仅 1 表有  → 自动归属 ✅                      │  │
│      │  ├─ 多表有    → AMBIGUOUS_COLUMN ⚠               │  │
│      │  └─ 都无      → UNKNOWN_COLUMN ❌                 │  │
│      │                                                  │  │
│      └─ 无限定 + 多表 + 部分元数据缺失                      │  │
│         └─ AMBIGUOUS_COLUMN（不猜）⚠                     │  │
│                                                         │  │
│     graph_builder:                                       │  │
│      merge_graphs("table",                               │  │
│        table_structure_graph,                            │  │
│        column_lineage_graph)                             │  │
│      → validate_graph（边源宿校验）                        │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                               │
│  3. 组装 AnalysisResult → JSON → 返回前端                      │
└───────────────────────────────────────────────────────────────┘

三、各 Service 调用关系

Controller
  │
  ├─ parse_sql(sql)
  │    └── complex_sql_guard.analyze_complex_sql(sql)
  │         ├─ PreflightChecker.check()
  │         ├─ DirtySqlPreprocessor.preprocess()
  │         ├─ SqlSegmenter.segment()
  │         ├─ SqlglotParserAdapter.parse() × 3 candidates
  │         └─ tree diagnostics (UDF/lateral walk)
  │
  ├─ analyze_cte_structure(tree)        ← CTE 路径
  │    └─ 提取 WITH 子句 → cte → table 依赖边
  │
  ├─ analyze_table_structure(tree)       ← 非-CTE 路径
  │    └─ _query_sources → table → result 边
  │
  ├─ resolve_column_lineage_names(tree, metadata)
  │    ├─ _detect_unsupported(has_metadata=True/False)
  │    ├─ _has_any_star → expand_star_items()
  │    │    ├─ _detect_star()  (exp.Star / Column(this=Star))     
  │    │    ├─ _expand_qualified_star()   (o.* → alias 解析)
  │    │    └─ _expand_unqualified_star()  (* → 全部源表展开)
  │    ├─ _simple_column_from_select_item()  (exp.Column / Alias)
  │    ├─ alias → table 解析 (alias_to_table mapping)
  │    ├─ metadata 字段验证: cols_for_table check
  │    └─ 多表消歧: tables_without_meta → AMBIGUOUS / candidates → 自动归属
  │
  └─ graph_builder
       ├─ build_column_lineage_graph(lineages)
       ├─ build_table_structure_graph(structure)
       ├─ build_cte_structure_graph(structure)
       ├─ merge_graphs(view_mode, *graphs)
       └─ validate_graph(graph)  ← 每条边 source/target 必存在

七、核心诊断码汇总

诊断码含义级别
SQL_PARSE_ERRORSQL 解析失败error
UNKNOWN_COLUMN字段不在任何已知表的元数据中warning
AMBIGUOUS_COLUMN多表都有同名字段，且无法消歧warning
UNKNOWN_TABLE_ALIAS表别名无法解析warning
SELECT_STAR_METADATA_REQUIREDselect * 需要元数据才能展开warning
METADATA_MISSING表在 SQL 中存在但元数据缺失warning
UNSUPPORTED_COMPLEX_QUERY复杂表达式暂不支持warning
UNSUPPORTED_SELECT_STAR旧版 select * 不支持（已废弃）warning
