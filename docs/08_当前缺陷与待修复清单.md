# 当前缺陷与待修复清单

更新时间：2026-06-05

## 目标

本清单用于收敛当前 `dev` 分支里已经确认的设计缺陷，并明确修复顺序。

本阶段遵循三条原则：

1. 优先修复会污染真实验收结果的问题。
2. 尽量复用现有状态和现有后端返回，不为了“看起来完整”快速膨胀代码。
3. 缺失能力先明确展示为 `unavailable`，不再用 mock 数据伪装真实结果。

## 当前问题分级

### P0：优先修复

1. 前端运行期混入 mock 数据
   - `SearchBar` 在后端未返回搜索结果时回退 `defaultOutputs/extraSearch`
   - `DetailPanel` 在后端未返回映射和定位信息时回退 mock `mappings/sourceLocations`
   - `Drawer` 在后端未返回诊断时回退 mock `diagnostics`
   - `selectors` 中的 `entityName/buildPathContext/diagnosticsForEntity` 对真实分析结果仍带有 mock 假设
   - 风险：复杂 SQL 验收时，页面可能展示“看起来合理但并非本次分析产生”的假结果

2. 元数据列查询前后端契约不一致
   - 前端使用 `?table=...`
   - 后端实际接收 `table_name`
   - 前端列展示读取 `column_name`
   - 后端返回字段名为 `name`
   - 风险：元数据列表页表能看到、列却始终为空

3. `partial` 结果被当成 `trusted`
   - 当前 UI 会放行搜索和定位交互
   - 风险：不完整结果被误解为高可信结果

4. `source_locations` 契约未打通
   - 后端模型有字段，但分析接口没有实际填充
   - 前端也没有把分析结果中的 `source_locations` 收入状态
   - 风险：`Locate SQL` UI 已存在，但真实链路未成立

### P1：下一阶段修复

1. 元数据仓储按裸 `table_name` 查询，缺少 `catalog/schema/table` 完整身份
2. 画布布局和重排依赖范围偏大，容易在大图场景下放大性能问题
3. 分析版本号和 schema 标识仍停留在旧阶段命名，容易造成认知混乱

## 本轮修复范围

本轮只做 `P0-1 前端运行期去 mock 污染`，不顺手扩到 `C08` 契约设计。

### 保留

1. 示例 SQL
2. 与分析结果无关的静态说明数据，例如 `milestones/snapshots`
3. 测试夹具中的 mock 图数据

### 去除

1. 搜索结果默认回退 mock 输出
2. 详情面板对 mock 映射、mock 定位信息的回退
3. 诊断面板对 mock 诊断的回退
4. `PathContext` 对 mock 路径和 mock mappings 数量的依赖
5. 运行期实体命名对 mock entity 表的依赖

## 修复验收标准

修复完成后应满足：

1. 用户点击 `Analyze` 后，页面只展示当前后端分析结果。
2. 如果后端没有返回搜索项、映射、定位或诊断，前端明确显示“暂无后端数据”。
3. 页面不再因为 mock fallback 展示与当前 SQL 无关的字段、映射或路径统计。
4. 不新增新的全局状态仓库，不为了补齐 mock 替代物而引入大体量中间模型。

## 后续建议顺序

1. 修 `P0-1`：前端运行期去 mock 污染
2. 修 `P0-2`：元数据列查询契约
3. 修 `P0-3`：`partial` 与 `trusted` 语义收敛
4. 修 `P0-4`：`source_locations` 前后端贯通
